# TSNoteCard 인라인 스레드 백엔드 문제 해결

## 문제 개요
TSNoteCard 페이지의 인라인 메모 스레드 기능에서 새로고침 시 작성자 정보가 '익명', '정보없음'으로 표시되는 백엔드 데이터 처리 문제

## 구체적 문제점
1. noteController.ts의 addInlineThread에서 authorName 설정 시 user.name, user.username을 찾고 있음
2. 실제 User 모델에는 nickname 필드만 존재함
3. 이로 인해 authorName이 제대로 설정되지 않아 새로고침 시 '익명'으로 표시됨

## 해결 방안
1. User 모델 스키마 확인 및 검증
2. noteController.ts의 authorName 설정 로직을 nickname 필드 사용하도록 수정
3. 기존 InlineThread 데이터 마이그레이션 (필요시)
4. 프론트엔드 데이터 핸들링 검증 및 개선
5. 종합 테스트 및 검증

## 기술적 제약
- 기존 데이터 손실 없이 수정해야 함
- 프로덕션 환경에 영향 최소화
- 사용자 경험 중단 없이 배포 가능해야 함

## 성공 기준
- 새로고침 후 작성자 이름이 정확히 표시됨
- '익명', '정보없음' 표시 문제 완전 해결
- 기존 데이터 무결성 유지
- 새로운 스레드 생성 시 정상 작동
