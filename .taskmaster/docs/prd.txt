# NotebookLM UI 기반 대시보드 구현 PRD

## 프로젝트 개요
NotebookLM의 UI 디자인을 참고하여 Habitus33 대시보드를 개선합니다. 기존 기능은 유지하면서 사용자 경험을 향상시키는 것이 목표입니다.

## 핵심 요구사항

### 1. 상단 헤더
- Habitus33 로고 (좌측)
- 설정 버튼 (우측)
- 사용자 프로필 이미지 버튼 (우측)

### 2. 메인 영역 (스크롤 없이 보이는 영역)
- 사용자 상태 메시지: "[사용자이름]님, 현재 [N]개의 메모를 만들었습니다."
- 핵심 액션 버튼: "+ 새로 만들기"
- 최근 메모 섹션:
  - 제목: "최근 메모"
  - 보기 전환 버튼 (카드/목록 뷰)
  - 정렬 드롭다운 (최신순 등)
  - 최근 메모 카드 3개
  - "나의 도서관" 진입 버튼 (작게)

### 3. 스크롤 영역
- 단권화 노트 섹션 (summary-notes 3개)
- Zengo / Zengo Myverse 진입 버튼
- 인지 분석 진입 버튼만 제공 (차트 표시 안함)

## 기술 스택
- Next.js 13+ (App Router)
- React 18+
- TypeScript
- Tailwind CSS
- 기존 API 엔드포인트 활용

## 제약사항
- 기존 기능 변경 금지
- 불필요한 새 기능 추가 금지
- NotebookLM UI 패턴 준수
- 모바일 반응형 필수

## 성공 기준
- NotebookLM과 유사한 사용자 경험
- 기존 기능 정상 동작
- 로딩 성능 유지
- 접근성 준수
