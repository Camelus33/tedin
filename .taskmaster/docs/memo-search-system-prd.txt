# 수험생 특화 메모카드 검색 시스템 PRD

## 프로젝트 개요
수험생이 작성한 수많은 메모카드를 효율적으로 검색하여 학습 진도 분석, 취약점 진단, 개인화된 학습 계획을 제공하는 고급 검색 시스템을 구축합니다.

## 비즈니스 목표
- 수험생의 학습 효율성 300% 향상
- 개인화된 학습 경험 제공
- 학습 진도 및 취약점 자동 진단
- 스마트 검색 제안으로 사용자 경험 개선

## 기술적 제약사항
- GPU 없음 (CPU 기반 처리만 가능)
- Render 백엔드 (서버리스 환경)
- MongoDB 데이터베이스
- Vercel 프론트엔드
- 최소한의 추가 인프라 비용

## 핵심 기능 요구사항

### 1. MongoDB Atlas Vector Search 통합
- 기존 MongoDB에 벡터 검색 기능 추가
- OpenAI Embeddings API를 활용한 의미론적 검색
- 하이브리드 검색 (키워드 + 벡터) 구현

### 2. 수험생 특화 검색 기능
- 학습 진도 검색 ("수학 미적분에서 복습 안 된 카드")
- 취약점 진단 검색 ("생각추가가 없는 메모카드")
- 시험 준비 검색 ("시험 2주 전 취약점 진단")
- 과목별, 태그별, 완성도별 필터링

### 3. 스마트 쿼리 제안 시스템
- 자주 사용하는 쿼리 패턴 학습
- 컨텍스트 기반 검색 제안
- 퍼플렉시티 스타일 채팅창 하단 제안

### 4. 실시간 학습 진단
- 학습 진도 분석 (완료/미완료 단원)
- 복습 우선순위 자동 계산
- 취약점 자동 진단 및 알림

## 기술 스택
- 백엔드: Node.js + Express (Render)
- 데이터베이스: MongoDB Atlas (Vector Search)
- 임베딩: OpenAI Embeddings API
- 프론트엔드: Next.js (Vercel)
- 캐싱: 메모리 기반 캐싱

## 성능 요구사항

- 동시 사용자: 100명
- 데이터 처리량: 1000건/분
- 정확도: 90% 이상

## 보안 요구사항

- 검색 히스토리 개인정보 보호

- 사용자 만족도 500% 향상
